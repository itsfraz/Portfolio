Habit Tracker | Full-Stack Motivation & Analytics
What it does:

Users can register and log in securely, then create habits with custom frequencies (daily, weekly, custom). Each habit tracks completion history, which feeds into a dashboard showing success rates, a GitHub-style streak heatmap, and completion charts. Users earn XP and badges as they stay consistent, and can share progress summaries.

How it's built:

The frontend is a React single-page app using Chart.js for data visualization and react-calendar-heatmap for streak tracking, styled with Bootstrap. The backend is a Node/Express REST API with MongoDB (via Mongoose) for data persistence and JWT for stateless authentication. Frontend and backend are deployed and hosted separately — React on Vercel, Express on Render — communicating over a REST API.

Challenges solved:
  • Diagnosed and fixed a CORS misconfiguration between the separately-hosted frontend and backend, including a subtle bug where a trailing slash in the allowed-origins list caused exact-match comparison to silently fail.
  • Caught and remediated an accidentally committed .env file containing live database credentials — rotated the exposed secrets and rebuilt the repo's environment-variable handling with a safe .env.example template.
  • Resolved a CI build failure on Vercel caused by Create React App treating ESLint warnings as build-breaking errors in production, fixing the underlying React hook dependency issues rather than suppressing them.

Wizard Diary | Interactive AI Magic Diary

Wizard Diary is a whimsical front-end experiment that reimagines an AI chatbot as an ancient, sentient diary. Instead of a standard chat window, the interface is a 3D leather-bound book: users click to swing the cover open, "write" a question onto the right-hand page, watch the ink physically dissolve into floating magic particles, and then read the wizard's typed response bleed onto the left page like wet ink drying.

Technical Highlights:
  • Advanced React & animation engineering: Built entirely in React 19 with Framer Motion, orchestrating a multi-stage state machine (IDLE → WRITTEN → VANISHING → THINKING → TYPING → READING → DONE) to sequence UI transitions precisely against async API calls.
  • Custom Canvas/WebGL-adjacent physics: Hand-rolled a particle system using the HTML5 Canvas API: per-character sine-wave drift, velocity, alpha decay, and glow rendering, with deliberate mobile-performance tuning.
  • Custom SVG filters for visual effects: Used an feGaussianBlur + feColorMatrix SVG filter to simulate "wet ink" bleeding as text renders character-by-character with staggered Framer Motion variants.
  • LLM API integration: Connects to the Groq Cloud API (llama-3.1-8b-instant) via fetch, with a custom system prompt persona, error handling, and a graceful offline/no-API-key simulation fallback so the demo always works.
  • Responsive, detail-obsessed CSS: Tailwind CSS v4 used for a richly layered aesthetic (radial gradients, inset shadows, brass-corner book hinges, animated magic circle) that adapts cleanly from mobile to desktop.
  • Modern tooling: Vite 8 build system, oxlint for linting, environment-variable-based API key management, and an audio manager utility for ambient sound cues.
